# Excel 合并功能设计（v0.1 定稿）

> 状态：已定稿（2026-07-21 与用户确认四个决策点），可开始实施。
>
> 已确认决策：① 直接在现有拆表工具中集成；② 字段不一致取并集 + 缺失留空 + 报告提示；③ 「自定义标题」= 自定义输出文件名；④ 「来源文件」列做成开关、默认关闭。

## 1. 这个功能是做什么的

举个例子：月底收到 5 个部门各自报上来的工时表，每个文件里都有「员工工时」「项目汇总」两个 sheet，列结构基本一样。现在要把它们合成一个文件：所有部门的「员工工时」摞在一起变成一个「员工工时」总表，「项目汇总」同理。这就是合并功能要做的事。

和现有拆分功能正好是一对逆操作：拆分是「一个文件按某列的值拆成多个文件」，合并是「多个结构一致的文件按 sheet 和字段对齐合成一个文件」。

## 2. 集成方式（已定稿：方案 A）

- **方案 A（已确认）：直接在现有拆表工具中扩展**。第一步「选择文件」页面顶部加一个功能模式切换：「拆分 / 合并」。选择合并后走合并专属的步骤流。后端新增 `merge_engine.py`、`merge_models.py`、`merge_routes.py`（挂在同一 Flask app 下），前端新增合并页面面板。
  - 理由：加密/解密（`encryption.py`）、输出目录选择、后台执行与进度、加密输出、桌面打包链路全部可以原样复用；用户只需要维护一个工具。
- 方案 B（已否决）：先做成独立脚本/独立页面，稳定后再合并进工具。多一次整合成本，且加密等逻辑要复制一份。

## 3. 合并规则

### 3.1 文件与 Sheet 对齐

- 批量添加多个 `.xlsx` 文件，文件列表支持删除和调整顺序（顺序决定数据在结果中的先后）。
- Sheet 按 **sheet 名** 对齐：所有文件里同名的 sheet 合并到输出文件的同名 sheet 中。
- 某个 sheet 只存在于部分文件：照常合并，结果中给出提示（哪些文件缺少该 sheet）。

### 3.2 字段对齐（核心）

- 不按列位置，而按 **表头字段名** 匹配。比如 A 文件「姓名」在第 1 列、B 文件「姓名」在第 2 列，也能正确对齐。
- 表头行可配置（默认第 1 行，和拆分功能一致，支持前若干行是标题的情况）。
- 字段不一致时的处理（**已定稿：取并集**）：以第一个文件的字段顺序为基准，其他文件多出的字段作为新列追加在表头末尾；某文件缺少的字段对应单元格留空，并在合并报告中逐条提示「文件 X 缺少字段 Y」。
- 表头匹配做基础规范化：去除首尾空格。大小写、全半角不做强制归一（避免误判，结果里给提示即可）。
- 空行（整行皆空）自动跳过（指追加的其他文件的数据行；第一个文件作为模板，其内容照抄不动，见 3.3）。

### 3.3 标题行与格式

> 2026-07-21 根据用户反馈重构为「模板照抄法」：用户要求输出严格保持输入表格的样式，且表头行只出现一次。

- 以**第一个输入文件为输出模板**（普通模式加载，公式、样式、合并单元格、冻结窗格等 openpyxl 往返能保住的内容全部保留），只删除未选中的 sheet。
- 每个选中 sheet 的表头行及之前所有行**照抄不动**，第一个文件自己的数据行也原地保留；其余文件的数据行流式读取后追加到末尾，追加时逐格复制样式（字体/填充/边框/对齐/数字格式/保护）与行高。
- 追加行的公式用 `openpyxl.formula.translate.Translator` 按行列偏移平移（相对引用随行变化，绝对引用不变）；模板自己的公式原文不动。
- 某个 sheet 在第一个文件中不存在时，从第一个包含它的文件逐格拷贝建出（openpyxl 不支持跨工作簿复制 sheet）。
- 用户可手动把某个 sheet 标记为「内容完全一致」：该 sheet 只保留模板中的一份，其他文件的同名 sheet 完全不读取（配合自动指纹去重开关，处理拆分输出中「完整保留」sheet 的重复合并问题）。
- 已知边界：条件格式、图片、数据验证、批注不保留。

### 3.4 公式平移与图表延伸（2026-07-21 补充）

- 追加行的公式按行列偏移平移（`Translator`）：同行相对引用随行变化，绝对引用不变，跨 sheet 的相对引用也会平移。多文件依次追加时每个文件的公式各自按源坐标平移，互不干扰（有测试锁定）。
- **风险警告**：追加公式中含「同 sheet、行号 ≤ 表头行的相对引用」（典型场景：引用标题区参数格的 `=B5*A1`）时，按 Excel 复制语义平移后很可能指错位置。引擎只检测常见 A1 形态（正则，排除跨 sheet 引用、绝对引用、函数名），检出后在结果警告里提示改用绝对引用，解析失败不警告（宁缺毋滥）。
- **图表数据范围延伸**：记录每个被追加 sheet 的「追加前末行 → 追加后末行」，合并完成后遍历所有工作表的图表 series 数据引用（val/cat/xVal/yVal），末行恰好等于追加前末行的完整区域延伸为新末行；单单元格引用、末行不匹配的部分区间、整列引用一律不动；任何异常只记警告不中断合并。identical sheet 未被追加，其图表引用自然不变。

### 3.5 多表区合并（2026-07-21 v0.4）

背景：拆分侧支持「多表区 sheet」（主表 + 空行分隔的若干小表，各有块表头）后，其拆分产物合并时小表行会被当普通数据追加，结构错乱。合并侧按对称设计支持，与拆分一致**手动标记**才启用（`MergeSheetConfig.has_sub_blocks`），未标记的 sheet 行为完全不变。

- **按需检测**：合并步骤 3 勾选后调 `/api/merge/sheet-blocks`，对每个输入文件做块检测（复用拆分侧的 `detect_blocks`，已移到 `excel_io.py` 共用），只读展示块结构；小表合并策略固定为「挂起重插 + 同表去重」，无需用户选择。
- **主表先合并**：每个输入文件先做块检测；模板裁剪到主表范围（删除小表区、分隔空行、幽灵行，整表指纹在裁剪前计算保持口径一致），其余文件只追加主表数据行，字段对齐/公式平移/空行跳过全部沿用主表逻辑。
- **小表挂起重插**：小表身份按**块表头内容**（规范化文本）识别，各文件同名小表视为同一个表；顺序以基准文件的出现顺序为准，模板没有的新小表追加在后。每个小表：先写 1 个空行分隔，再写块表头（取第一个含该表的文件，带样式），再按文件顺序写各文件该小表的数据行（按原列位照抄、逐格样式、公式按行平移、整行全空跳过）。
- **块级指纹去重**：同一小表在多文件内容完全相同时（典型：拆分侧 keep 策略产物）只保留一份并记警告；不同文件有不同数据行时按文件顺序拼接（拆分产物的正常形态）。
- **图表避让顺序**：主表合并 → 小表插入 → 图表数据范围延伸（只针对主表数据区）→ 图表锚点避让（锚点落在主表之下的，整体下移到最终内容末行之后；主表区内的锚点不动）。
- 拆分→合并往返可还原原表结构（合计行除外，其拆分键为空在拆分时已被丢弃）。

## 4. 密码处理

- 添加文件后，后端逐个用现有 `is_encrypted()` 检测。
- 存在加密文件时，界面提示输入 **一个统一密码**，后端用它逐个尝试解密所有加密文件（复用 `decrypt_file()`）。
- 解密失败的文件单独列出，用户可以选择：
  - 为该文件单独补一个密码（覆盖统一密码）；或
  - 从列表中移除该文件后继续。
- 密码只用于本次读取，不保存。

## 5. 输出设置

- 输出为 **单个 `.xlsx` 文件**，内含合并后的全部 sheet。
- 保存地址：默认与 **第一个输入文件同目录**（符合「默认和输入表格同一路径」要求），可通过「选择文件夹」按钮改（复用现有 `choose_directory`）。
- 文件名（即「自定义标题」，**已定稿：仅指输出文件名**）：允许自定义，默认模板 `{第一个文件名}_合并结果.xlsx`，不在 sheet 内额外加大标题行。
- 可选「来源文件」列（**已定稿：提供开关、默认关闭**）：开启后每个 sheet 末尾加一列「来源文件」，记录该行来自哪个输入文件，便于追溯。
- 可选加密输出：勾选后设置密码，输出文件用现有 `encrypt_file()` 加密，与拆分功能行为一致。

## 6. 界面流程（合并模式）

1. **选择功能模式**：拆分 / 合并。
2. **添加文件**：多选添加，列表展示（文件名、大小、是否加密），支持移除、上下调序；检测到加密文件时进入密码输入。
3. **配置 Sheet**：列出所有文件出现过的 sheet 名（勾选参与合并的 sheet），每个 sheet 可设表头行；提供「字段匹配检查」预览——展示每个 sheet 的并集字段、各文件缺失/多余字段，让用户执行前就能发现问题。
4. **输出设置**：保存路径、文件名、（可选）大标题、（可选）来源文件列、（可选）加密与密码。
5. **执行与结果**：后台执行 + 进度条（复用现有机制），完成后显示「打开文件 / 打开所在文件夹」，以及合并报告（每个 sheet 合并了多少行、来自几个文件、警告列表）。

## 7. 技术实现要点

- 新增模块：
  - `src/excel_splitter/merge_models.py` — `MergeJob` / `MergeSheetConfig` / `MergeResult` / `MergeSummary`，风格对齐现有 `models.py`（frozen dataclass + validate）。
  - `src/excel_splitter/merge_engine.py` — `MergeEngine.execute(job, progress_callback)`。
  - `src/excel_splitter/merge_planning.py` — 字段匹配检查（并集字段、缺失/多余报告），供第 3 步预览和执行复用。
  - `src/excel_splitter/web/merge_routes.py` — `/api/merge/*` 接口；前端 `index.html`/`app.js`/`app.css` 增加合并面板。
- 性能：读取一律 `read_only=True` 流式扫描（吸取拆分功能的教训，不缓存非 read_only 工作簿）；输出用 `write_only=True` 工作簿，万级行数无压力。
- 解密后的文件按现有拆分的做法落到临时目录（`.decrypted.xlsx`），任务结束随上传目录清理。
- 测试：
  - `tests/test_merge_models.py` — 配置校验。
  - `tests/test_merge_engine.py` — 字段并集/缺失留空/表头行偏移/空行跳过/顺序保持/加密输入/加密输出。
  - `tests/test_merge_routes.py` — 上传、密码、字段检查预览、执行接口。
- 打包：`msoffcrypto` 已在依赖中，PyInstaller spec 无需新增隐式导入（实施时验证）。
